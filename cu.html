Tâche 3

Cas d'utilisation : S'inscrire en tant qu'utilisateur

But : Permettre de devenir un utilisateur, ce qui donne accès aux robots.

Acteur : Utilisateur

Préconditions :

  • Fournir les données personnelles.
  • Fournir le nom de la compagnie.

Scénario principal :

  1. Choisir "créer un compte".
  2. Fournir les données personnelles.
  3. Créer un mot de passe.
  4. Confirmer le mot de passe.
  5. Vérifier si le mot de passe est valide.
  6. Vérifier si le mot de passe est disponible.
  7. Enregistrer le mot de passe.
  8. Créer un nouveau compte dans le système.
  9. Afficher un message de succès à l'utilisateur.

Scénario alternatif :

4.1. Si la confirmation du mot de passe est incorrecte.

4.2. Afficher un message d'erreur "le mot de passe de confirmation est invalide".

4.3. Reprendre à partir de l'étape 4 du scénario principal.

Postconditions :

  • Enregistrer l'utilisateur.
  • Sauvegarder un numéro de référence pour l'utilisateur dans le système.

Cas d'utilisation : Enregistrer un robot

But : Enregistrer le robot dans la flotte de l’utilisateur.

Acteur principal : L'utilisateur

Préconditions :

  • Fournir un nom et le type du robot.
  • Fournir le numéro de série donné par le fournisseur.

Scénario principal :

  1. Choisir « enregistrer un robot ».
  2. Fournir le nom et le type du robot.
  3. Fournir le numéro de série.
  4. Valider le numéro de série.
  5. Ajouter le robot dans la flotte.
  6. Afficher un message de succès.

Scénario alternatif :

4.1. Si le numéro de série fourni n’est pas valide.

4.2. Revenir à l’étape 3 du scénario principal.

Postconditions :

  • Le robot est enregistré dans la flotte.

Cas d'utilisation : Acheter des composantes

But : Acheter les composantes nécessaires pour construire ou réparer le robot.

Acteur principal : L'utilisateur

Acteur secondaire : Fournisseur

Préconditions :

  • Avoir les fonds nécessaires.
  • Avoir un fournisseur.

Scénario principal :

  1. Aller sur le site du fournisseur.
  2. Rentrer dans le compte.
  3. Choisir l’option « commander des composantes ».
  4. Ajouter les composantes désirées au panier.
  5. Choisir la méthode de paiement.
  6. Payer.
  7. Validation du paiement.
  8. Afficher un message de succès.
  9. Attendre la commande.
  10. Recevoir les composantes.

Scénario alternatif :

6.1. Si les fonds dans la carte ne sont pas suffisants.

6.2. Afficher un message d’erreur « fonds insuffisants ».

Postconditions :

  • Les composantes sont livrées avec factures.
  • Un numéro de transaction est créé pour sauvegarder l’historique des transactions.

Cas d'utilisation : Contrôle des mouvements

Acteur principal : L'utilisateur

Acteur secondaire : Le robot

But : Contrôler les mouvements du robot en lui donnant des ordres de déplacement.

Préconditions :

  • Le robot doit avoir des roues ou des hélices pour pouvoir se déplacer.

Scénario principal :

  1. L'utilisateur essaie de se connecter au robot.
  2. Le robot se connecte à l'utilisateur.
  3. L'utilisateur reçoit une confirmation.
  4. L'utilisateur contrôle le déplacement du robot en donnant des ordres avec son propre mouvement.
  5. Le robot détecte le mouvement de l'utilisateur.
  6. Le robot utilise ses roues ou hélices pour se déplacer.
  7. Le robot attend les prochains ordres de l'utilisateur.

Scénario alternatif :

1.1. Si le robot n'arrive pas à se connecter à l'utilisateur.

1.2. Afficher un message d'échec à l'utilisateur.

1.3. Retourner à l'étape 1.

Postconditions :

  • Le robot revient dans son état de repos.

Cas d'utilisation : Création des actions

But : Permettre de donner des ordres mécaniques aux robots.

Acteur principal : L'utilisateur

Acteur secondaire : Le robot

Préconditions :

  • Les composantes nécessaires pour exécuter l’action doivent être disponibles.

Scénario principal :

  1. Choisir l’option « créer une action ».
  2. Choisir le robot désiré.
  3. Choisir le type d’action.
  4. Validation des composantes.
  5. Donner les spécificités de l’action.
  6. Le robot enregistre l’action.
  7. Revenir au menu.

Scénario alternatif :

4.1. Si les composantes sont manquantes.

4.2. Afficher le message d’erreur « ces pièces sont manquantes, veuillez les installer avant de continuer ».

4.3. Reprendre à l’étape 4 du scénario principal.

Postconditions :

  • Le robot enregistre les actions et les exécute quand on les appelle.

Cas d'utilisation : Gestion des tâches

But : Donner des tâches au robot en fonction de leur disponibilité et compétences au moment souhaité sans l’intervention de l’utilisateur.

Préconditions :

  • Le robot doit être connecté à l’utilisateur et disponible à être utilisé.

Scénario principal :

  1. L’utilisateur choisit un robot pour exécuter une tâche.
  2. L’utilisateur choisit la tâche et l’heure de la tâche.
  3. L’utilisateur envoie l’ordre au robot.
  4. Le robot s’assure qu’il est capable de réaliser la tâche.
  5. Le robot regarde s'il y a d'autres tâches à faire et regarde quelle transition est possible.
  6. Le robot donne une confirmation à l'utilisateur pour confirmer.

Scénario alternatif :

4.1. Si le robot n’est pas assez compétent pour réaliser la tâche.

4.2. Le robot envoie un message d’erreur à l'utilisateur pour lui dire qu’il n’est pas disponible.

4.3. Retourner à l'étape 4.

Postconditions :

  • Le robot revient à son état normal.

Cas d'utilisation : Afficher l’état de mon robot

But : Afficher les données de mon robot pour permettre de savoir son état.

Acteur principal : L’utilisateur

Acteur secondaire : Le robot

Préconditions :

  • Avoir les bons capteurs installés.
  • Être connecté à la flotte.

Scénario principal :

  1. Se connecter au compte.
  2. Accéder à la flotte.
  3. Choisir le robot désiré.
  4. Choisir l’option « afficher l’état ».
  5. Validation des capteurs.
  6. Afficher les données demandées.
  7. Fermer la fenêtre des données quand satisfait.

Scénario alternatif :

5.1. Si les capteurs sont manquants.

5.2. Afficher le message d’erreur : « Les capteurs nécessaires sont manqués, veuillez les installer avant de continuer ».

5.3. Le programme reprend à l’étape 5 du scénario principal.

Postconditions :

  • Le robot revient à son état prêt.

Cas d'utilisation : Afficher les métriques de ma flotte

But : Afficher la condition de mes robots.

Acteur principal : L’utilisateur

Préconditions :

  • Avoir les bons capteurs installés.
  • Être connecté à la flotte.

Scénario principal :

  1. L’utilisateur clique sur un bouton pour afficher les détails de la flotte.
  2. L’utilisateur vérifie si tous les robots ont un capteur.
  3. Le système retourne les détails de la flotte.
  4. L’utilisateur peut voir les activités et informations des robots.

Scénario alternatif :

1.1. Si certains robots ne sont pas bien connectés au capteur.

1.2. L’utilisateur s’assure que tous les robots sont connectés.

1.3. Retour à l’étape 1.

Postcondition :

  • Le robot revient à son état normal.

Cas d'utilisation : Gestion de problèmes

But : Détection d'erreurs et gérer ces erreurs en fournissant des alertes et messages d'erreur.

Acteur Principal : L'utilisateur

Précondition :

  • Définition des critères d'erreurs.
  • Système de surveillance, que ce soit des capteurs, système interne, etc.

Scénario principal :

  1. Détection de l'erreur.
  2. Enregistrement de l'erreur.
  3. Génération d'une alerte.
  4. Affichage du message d'erreur.
  5. Initiation des actions correctives.
  6. Enregistrement des actions.
  7. Résolution de l'erreur.
  8. Analyse post-incident.

Scénario alternatif :

7.1. N'arrive pas à régler l'erreur.

7.2. Générer un rapport.

7.3. Arrêt du programme.

Postcondition :

  • Alerte envoyée.
  • Résolution d'erreur.

Cas d'utilisation : S'inscrire comme fournisseur

But : Permet de devenir un fournisseur et d'accéder à la vente de robots et de leurs composants.

Acteur Principal : Le fournisseur

Préconditions :

  • Posséder une entreprise.
  • Disposer d'un nom, d'une adresse, d'une adresse courriel et d'un numéro de téléphone.
  • Fournir une capacité de fabrication.

Scénario principal :

  1. Choisir « créer un compte ».
  2. Fournir les données personnelles.
  3. Créer un mot de passe.
  4. Confirmer le mot de passe.
  5. Vérifier si le mot de passe est valide.
  6. Vérifier si le mot de passe est disponible.
  7. Enregistrer le mot de passe.
  8. Créer un nouveau compte chez le système.
  9. Afficher un message de succès fournisseur.

Scénario alternatif :

4.1. Afficher un message d’erreur « le mot de passe de confirmation est invalide ».

4.2. Reprendre à partir de l’étape 4.

Postconditions :

  • Enregistrer le fournisseur.
  • Sauvegarder un numéro de référence pour le fournisseur dans le système.
