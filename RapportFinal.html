Tâche 5 Bonus (Interface & Github Actions)

Nous avons fait une interface graphique avec JavaFx pour notre application.

Nous savons exécuter des GitHub Actions, mais la mise en œuvre de nos tests s'avère trop complexe. Malgré toutes nos tentatives, nous n'avons pas réussi à contourner les difficultés liées à la simulation de l'interface graphique, même avec Xvfb. Voici comment fonctionnent nos tests :

La méthode de test appelle une méthode du contrôleur pour accéder à certaines méthodes privées, en lui fournissant les arguments nécessaires (par exemple, pour la méthode createAccountClient, nous fournissons toutes les informations d'un utilisateur). Cette méthode écrit sur une interface temporaire et invoque ensuite la véritable méthode qui effectue l'écriture dans la base de données en utilisant les données de cette interface temporaire. Par la suite, la méthode de test lit les informations dans la base de données et vérifie qu'elles correspondent aux données initialement fournies via la méthode intermédiaire. Si le processus échoue, aucune donnée ne sera enregistrée ; de plus, toute erreur dans l'écriture sera également détectée. Enfin, le test supprime l'utilisateur de la base de données, car cet utilisateur a été créé uniquement pour les besoins du test.

En fin de compte, nous avons créé une classe SimpleTest pour démontrer notre capacité à utiliser le fichier YAML afin de déclencher des GitHub Actions à chaque push dans la branche main. Nous avons conservé une longue liste d'instructions qui étaient supposées être utilisées mais ne fonctionnaient pas comme trace de notre travail.
